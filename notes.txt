//      -------  02 --------    //

Variables:

1. se recomienda usar snake case
2. para variables privadas se inicializan con "_"
3. para constantes se usa mayúsculas

HOF:

1. Basicamente son funciones que reciben una función como argumento o que retornan una función

Functions (def)

1. Cuando se declara una función a lo que se le pasa por dentro de los parentesis, se le debe decir parametros y al momento de llamar esta función rellenando los parametros, esto se llama argumentos

Todo esto se ha definido por la comunidad, como tal no es algo que venga con lenguaje

CLASES Y OBJETOS:

1. Al momento de crear una clase, la primera letra va en mayúsculas
2. Al momento de crear los atributos (mapa/esquema) de la calase, se debe hacer sobre una función llamada __init__
2. Para crear un atributo/metodo protegido, se debe inicializar el atributo/metodo con "_", es el estandar
3. Para crear un atributo/metodo privado, se debe inicializar el atributo/metodo con "__", en este caso, python si interpreta que es un valor privado
4. cuando se almacena la clase en una variable, a eso se le llama instancia
5. el classmethod funciona para cambiar atributos a nivel de clase
6. el staticmethod funciona para ejecutar funcionalidades a nivel de clase o instancia

//      -------  Generales --------    //

1. Todo en python es un objeto
2. Las tuplas son inmutables, sirve para guardar valores fijos
3. Los sets a diferencia de los arrays y tuplas, no están organizados, o sea, no tienen indice, tampoco permite duplicados (los elimina) 
4. DTL (Django temaplte language) basicamente es un HTML con funcionalidades adicionales

//      -------  Templates --------    //

1. Crear una carpeta templates dentro de nuestra aplicación, ideal crear una subcarpeta con el nombre de la app para que django no sobrescriba otros templates con el mismo nombre
2. registrar nuestra app en el settings.py en la constante INSTALLED_APPS o registrar la ruta en TEMPLATES.DIRS
3. Usar el método render para renderizar el template desde la vista


//      -------  Instalación django comandos --------    //

1. Ideal crear el entorno virtual para encapsular nuestro proyecto con: /opt/homebrew/bin/python3 -m venv {Nombre entorno virtual}
2. Para activar el VENV se hace con: source {Nombre entorno virtual}/bin/activate
3. Para desactivar el entorno virtual se usa deactivate
4. Para inicializar un proyecto de 0 es con el siguiente comando django-admin startproject nombre_proyecto y en caso de solo querer inicializarlo con la carpeta de config poner un '.' al final
5. Para correr el proyecto se usa: python3 manage.py runserver
6. Para crear apliaciones dentro de mi proyecto se usa: python3 manage.py startapp nombre_aplicacion
7. Para migraciones python3 manage.py migrate  
8. Paso previo antes de crear las migraciones python3 manage.py makemigrations {{ app_name }}
9. Para ver las migraciones completadas/pendientes  python3 manage.py showmigrations

//      -------  Modelos --------    //

1. Entidad, atributo y relación
1.1 entidad = objeto del mundo real representado en el sistema (libros, autores etc...)
1.2 atributo = caracteristica o propiedad de la entidad (por ejemplo color del libro, total de páginas etc...)
1.3 relación = asosiación entre entidades (por ejemplo un libro tiene un autor)

2. Modelos conceptuales, logicos y físicos
2.1 conceptual = enfocado en representar las entidades y relaciones principales sin preocuparse por detalles técnicos
2.2 lógico = traduce el modelo conceptual a una estructura compatible con bases de datos relacionales
2.3 físico = implementación en un motor de base de datos

3.Bases de datos relacionales

3.1 almancena datos en una base de datos estructurada, relacionando los datos mendiante llaves primarias y foraneas


//      -------  ORM --------    //

4. ORM (Object relation mapping)

4.1 Es una herramienta que permite trabajar con bases de datos usando clases u objetos en un lenguaje de promación en lugar de SQL

4.2 COMANDOS

4.2.1 Para ejecutar comandos del ORM desde consola se debe ingresar primero a la terminal de DJango, el comando para ingresar es python3 manage.py shell
4.2.2 Para hacer de la terminal un poco más amigable, podemos instalar el paquete "ipython"
4.2.3 CREAR REGISTROS DE MANERA DIRECTA - Para crear registros se hace con CLASS.objects.create(params="")
4.2.3 CREAR REGISTROS DE MANERA INDIRECTA - Para crear registros se hace con INSTANCIA + INSTANCIA.SAVE()
4.2.4 CREAR REGISTROS DE MANERA MASIVA - Para crear registros se hace con CLASS.objects.bulk_create([ INSTANCIA(params="") ])
4.2.5 CONSULTAR Y SI NO EXISTE CREAR - Se hace de la siguiente manera CLASS.objects.get_or_create(query, defaults={ ...params })
4.2.6 ACTUALIZAR Y SI NO EXISTE CREAR - Se hace de la siguiente manera CLASS.objects.update_or_create(query, defaults={ ...params })

4.2.7 OBTENER TODOS LOS ELEMENTOS - Se hace de la siguiente manera CLASS.object.all()
4.2.8 OBTENER PARAMETRO ESPECIFICO - Se hace de la siguiente manera CLASS.object.get(param="")
4.2.9 OBTENER PRIMER REGISTRO - Se hace de la siguiente manera CLASS.object.first()
4.2.10 OBTENER ÚLTIMO REGISTRO - Se hace de la siguiente manera CLASS.object.last()
4.2.11 ORDENAR QUERY ASC - Se hace de la siguiente manera CLASS.object.order_by('param', 'other_param')
4.2.12 ORDENAR QUERY DESC - Se hace de la siguiente manera CLASS.object.order_by('-param')
4.2.13 FILTRAR POR PARAMETROS - Se hace de la siguiente manera CLASS.object.filter(param="") tener en cuenta que por defecto viene el __exact en cada parametro filtrado
Para filtrar por otros tipos de casos se debe hacer de la siguiente manera CLASS.objects.filter(param__case=""), entre esos casos están:
    4.2.13.1 param__contains
    4.2.13.2 param__iexact = insensitive, o sea, ignora mayúsculas y minúsculas
    4.2.13.3 param__startswith
    4.2.13.4 param__endswith
4.2.14 OBTENER QUERY SQL - INSTANCIA.query
4.2.15 FILTRAR POR DICCIONARIO DE DATOS - Se hace de la siguiente manera CLASS.objct.filter(param__in=[value,value,value])
4.2.16 FILTRAR POR MAYOR QUE O MAYOR E IGUAL - Se hace de la siguiente manera CLASS.objct.filter(param__gt=value) o CLASS.objct.filter(param__gte=value)
4.2.17 FILTRAR POR MENOR QUE O MENOR E IGUAL - Se hace de la siguiente manera CLASS.objct.filter(param__lt=value) o CLASS.objct.filter(param__lte=value)
4.2.18 FILTRAR POR FECHA EN CAMPOS DE TIPO DATE - Se hace de la siguiente manera CLASS.objct.filter(param=date(YYYY,MM,DD))
4.2.19 FILTRAR POR FECHA EN OTROS TIPOS DE CAMPOS - Se hace de la siguiente manera CLASS.objct.filter(param__date=date(YYYY,MM,DD))
4.2.20 FILTRAR POR AÑO - Se hace de la siguiente manera CLASS.objct.filter(param__year=YYYY)
4.2.21 FILTRAR POR MES - Se hace de la siguiente manera CLASS.objct.filter(param__month=MM)
4.2.22 FILTRAR POR DÍA - Se hace de la siguiente manera CLASS.objct.filter(param__day=DD)
4.2.23 FILTROS ENCADENADOS - Se hace de la siguiente manera CLASS.objct.filter(param=value).filter(param=value)
4.2.24 FILTROS EXCLUIDOS - Se hace de la siguiente manera CLASS.objct.filter(param=value).exclude(param=value)
4.2.25 FILTROS LIMITADO - Se hace de la siguiente manera CLASS.objct.filter(param=value)[init:end]
4.2.26 FILTROS Q - Se hace de la siguiente manera CLASS.objct.filter(Q(param=value) & Q(param=value) | (param=value))
4.2.27 ELIMINAR DATOS - Se hace obtiene un items y luego se usa el metodo delete: item.delete()
4.2.28 ELIMINAR DATOS - Se hace obtiene un items y luego se usa el metodo delete: item.delete()
4.2.29 TIPO DE ON_DELETE CUANDO LOS ITEMS TINES RELACIONES:
    4.2.29.1 CASCADE: Elimina todos los elemetnos relaciones 
    4.2.29.2 PROTECT: No permite eliminar elementos si existe una relación
    4.2.29.3 SET_NULL: Permite borrarlo y todas las relaciones se ponen en null
    4.2.29.4 SET_DEFAULT: Usa un valor por defecto
    4.2.29.4 DO_NOTHING: No hace nada, puede lanzar un error de integridad entre relaciones
4.2.30 AGGREGATE: Se usa asi: CLASS.objects.aggregate(any_name_result=Count('column')) también permite usar Min, Max, Sum etc...
4.2.30 ANNOTATE: Se usa asi: CLASS.objects.annotate(any_name_result=Count('related_name_of_model')) también permite usar Min, Max, Sum etc...
